## 通用设计原则（适用于所有方案）
- 核心闭环：**输入（阅读/对话/听力）→ 采集（词/句）→ 练习（复习/口语/写作）→ 反馈（纠错/数据）**
- Chat 是“教练”：能随时解释、出题、纠错、生成例句、规划学习。
- 词汇是“资产”：每个词/短语都沉淀到个人词库，并可追溯来源（哪段对话/文章）。
- 数据可视：用日历、连胜、掌握度、薄弱点来驱动持续学习。

## 评审建议
- 先选定 1 个核心场景：口语提升 / 考试备考 / 听力精听 / 词汇记忆。
- 再看信息架构是否最短路径：从打开 App 到开始学习 ≤ 2 步。
- 最后评估可扩展性：是否方便加入更多节点（例如 ASR/TTS/Grammar Checker）。
