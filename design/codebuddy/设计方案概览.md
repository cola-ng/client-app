# CodeBuddy 英语学习软件设计方案

## 设计理念

### 核心目标
创造一个**低压力、高沉浸、个性化**的英语学习环境，让用户在AI陪伴下自然提升口语能力。

### 设计原则
1. **零羞耻感** - 界面温馨，AI像朋友般对话，鼓励大胆开口
2. **场景化沉浸** - 通过视觉+音效带入真实场景（咖啡厅、机场、面试等）
3. **渐进式反馈** - 错误不是"红色警告"，而是温和的提示和改进建议
4. **学习流融合** - 复习计划、错题本自然融入对话，不打断用户思路
5. **社交化辅助** - 英语助手让真实交流更自信，AI像"同声传译"

### 与现有方案对比
| 方案 | CodeBuddy优势 |
|------|---------------|
| 方案1（移动端） | 更适合桌面端深度学习，场景化更丰富 |
| 方案3（卡片流） | AI主动引导更强，场景沉浸感更深 |
| 方案4（沉浸式） | 复习更智能，口音/场景更丰富，社交辅助更实用 |

## 设计方案详细说明

### 方案A：场景沉浸式主界面（推荐）

#### 特点
- **场景化主题**: 每天一个真实场景（咖啡厅、机场、面试、购物等）
- **AI角色扮演**: 不仅是教练，更是场景中的"真人"
- **温和纠错**: 错误提示像朋友的建议，而非批改
- **浮动提示**: 智能推荐不打断对话流

#### 页面元素

**1. 场景预览区（顶部）**
```
┌────────────────────────────────────────────────────────┐
```

## 设计创新总结

### 比现有方案的提升
1. **场景化** - 更强沉浸感，不只是对话，而是"真实环境"
2. **AI角色化** - Emma不只是工具，而是学习伙伴
3. **智能复习** - AI自动生成计划，不需要用户规划
4. **社交辅助** - 填补了"真实练习"的空白
5. **情绪感知** - 学习状态可视化，AI调整策略
6. **电影学习** - 从经典中学习，更有代入感

### 人性化体现
- 温和的纠错，不是"你错了"，而是"试试这个"
- 鼓励为主，提示为辅
- 学习成就可视化，即时正向反馈
- 错误记录成"学习点"而非"错误列表"
